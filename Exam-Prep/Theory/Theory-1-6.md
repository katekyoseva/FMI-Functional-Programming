# Въведение в Haskell

## Въведение

Функционалното програмиране (ФП) е парадигма, в която програмите се изграждат чрез **композиция на функции**. За разлика от императивното програмиране (C, Java), тук няма променливи, цикли или странични ефекти - всичко е **изрази**, които се **оценяват** до стойности.

**Haskell** е чисто функционален, статично типизиран, мързеливо оценяващ език.

---

## 1. GHCi - интерактивна среда

### Инсталация

Използвайте [GHCup](https://www.haskell.org/ghcup/) за инсталация на GHC (компилатор) и GHCi (интерактивна среда).

### Стартиране

```bash
ghci
```

GHCi е REPL (Read-Eval-Print Loop) - въвеждате израз, получавате резултат:

```haskell
ghci> 2 + 3
5
ghci> "Hello" ++ " World"
"Hello World"
```

### Зареждане на файл

```bash
ghci> :load MyFile.hs    -- или :l MyFile.hs
ghci> :reload             -- или :r (след промяна на файла)
ghci> :type expression    -- или :t (показва типа)
ghci> :info SomeType      -- или :i (информация за тип)
ghci> :quit               -- или :q (изход)
```

| --------- | ------------------------- | ------------------- |

| `Integer` | Цяло число (неограничено) | `123456789012345`   |

```haskell
ghci> 2 + 3          -- 5
ghci> 10 - 4         -- 6

ghci> 10 / 3         -- 3.3333...  (дробно деление)

ghci> 3 < 5     -- True
ghci> 3 >= 3    -- True
```

### Булеви оператори

```haskell
ghci> True && False   -- False

ghci> not True        -- False
```

## 4. Условни изрази

### `if-then-else`

В Haskell `if` е **израз** (връща стойност), не оператор:

```haskell
ghci> if 3 > 2 then "yes" else "no"
"yes"
```

> ⚠️ `else` е **задължителен**! Всеки клон трябва да връща стойност от **един и същ тип**.

```haskell
absolute :: Int -> Int
absolute x = if x >= 0 then x else -x
```

---

## 5. Дефиниране на функции

### Прост синтаксис

```haskell
-- Типова сигнатура (незадължителна, но препоръчителна)
double :: Int -> Int
double x = x * 2

-- Функция с два аргумента
add :: Int -> Int -> Int
add x y = x + y
```

> 💡 Няма скоби около аргументите и няма запетаи между тях! `add(x, y)` е грешно - правилно е `add x y`.

### Извикване

```haskell
ghci> double 5      -- 10
ghci> add 3 4       -- 7
ghci> add (double 2) 3  -- 7  (скоби само за групиране)
```

## 6. Типови сигнатури

```haskell
functionName :: ArgType1 -> ArgType2 -> ReturnType
```

Примери:

```haskell
isEven :: Int -> Bool
isEven x = x `mod` 2 == 0

hypotenuse :: Double -> Double -> Double
hypotenuse a b = sqrt (a^2 + b^2)
```

> 💡 `->` е дясно-асоциативен: `Int -> Int -> Int` е `Int -> (Int -> Int)`. Това е свързано с **currying** - ще го разгледаме по-късно.

## 7. `let` и `where`

### `where` - локални дефиниции след тялото

```haskell
bmi :: Double -> Double -> String
bmi weight height
  | index < 18.5 = "Поднормено"
  | index < 25.0 = "Нормално"
  | index < 30.0 = "Наднормено"
  | otherwise     = "Затлъстяване"
  where
    index = weight / height ^ 2
```

### `let ... in` - локални дефиниции преди израза

```haskell
cylinderArea :: Double -> Double -> Double
cylinderArea r h =
  let baseArea = pi * r ^ 2
      sideArea = 2 * pi * r * h
  in  2 * baseArea + sideArea
```

# Образци, гардове и кортежи

## Въведение

В Haskell функциите се дефинират чрез **pattern matching** (съпоставяне по образци) и **guards** (гардове/стражи). Това е по-декларативен подход от `if-then-else` - описваме **какво е** резултатът за различни форми на входа, вместо **как** да го пресметнем.

---

## 1. Pattern Matching - образци

### Литерални образци

```haskell
dayOfWeek :: Int -> String
dayOfWeek 1 = "Понеделник"
dayOfWeek 2 = "Вторник"
dayOfWeek 3 = "Сряда"
dayOfWeek 4 = "Четвъртък"
dayOfWeek 5 = "Петък"
dayOfWeek 6 = "Събота"
dayOfWeek 7 = "Неделя"
dayOfWeek _ = "Невалиден ден"
```

Проверката върви **отгоре надолу** - първият съвпадащ образец „печели".

### Видове образци

| Образец      | Описание                                  | Пример             |
| ------------ | ----------------------------------------- | ------------------ |
| Литерал      | Точна стойност                            | `0`, `True`, `'a'` |
| Променлива   | Съвпада с всичко, свързва стойността      | `x`, `name`        |
| Wildcard `_` | Съвпада с всичко, **игнорира** стойността | `_`                |

```haskell
not' :: Bool -> Bool
not' True  = False
not' False = True

isZero :: Int -> Bool
isZero 0 = True
isZero _ = False
```

> ⚠️ Ако пропуснете случай, ще получите runtime грешка. Компилаторът може да предупреди с `-Wall`.

---

## 2. Guards - гардове

Guards са „условия" към pattern matching - като `if-else` верига, но по-четима:

```haskell
bmiCategory :: Double -> String
bmiCategory bmi
  | bmi < 18.5 = "Поднормено тегло"
  | bmi < 25.0 = "Нормално тегло"
  | bmi < 30.0 = "Наднормено тегло"
  | otherwise   = "Затлъстяване"
```

### Синтаксис

```haskell
functionName arg1 arg2
  | condition1 = result1
  | condition2 = result2
  | otherwise  = defaultResult
```

> 💡 `otherwise` е просто `True` - хваща всичко останало.

### Пример: абсолютна стойност

```haskell
myAbs :: Int -> Int
myAbs x
  | x >= 0    = x
  | otherwise  = -x
```

---

## 3. Кортежи (Tuples)

Кортежът е **фиксирана** поредица от стойности с **потенциално различни типове**:

```haskell
-- Двойка (pair)
point :: (Double, Double)
point = (3.0, 4.0)

-- Тройка (triple)
person :: (String, Int, Bool)
person = ("Иван", 22, True)
```

### `fst` и `snd` - само за двойки

```haskell
ghci> fst (1, 2)
1
ghci> snd (1, 2)
2
```

### Pattern matching на кортежи

```haskell
addPairs :: (Int, Int) -> (Int, Int) -> (Int, Int)
addPairs (x1, y1) (x2, y2) = (x1 + x2, y1 + y2)

getName :: (String, Int) -> String
getName (name, _) = name
```

---

## 4. `where` с деструктуриране

```haskell
initials :: String -> String -> String
initials first last = [f] ++ "." ++ [l] ++ "."
  where
    (f:_) = first   -- деструктурира списъка, взима първия елемент
    (l:_) = last
```

---

## 5. Комбиниране на patterns и guards

Можем да комбинираме pattern matching с guards:

```haskell
describeList :: [a] -> String
describeList xs
  | length xs == 0 = "Празен списък"
  | length xs == 1 = "Едноелементен списък"
  | otherwise      = "Списък с много елементи"
```

Или с pattern matching:

```haskell
describeList' :: [a] -> String
describeList' []  = "Празен списък"
describeList' [_] = "Едноелементен списък"
describeList' _   = "Списък с много елементи"
```

> 💡 Вторият вариант е по-идиоматичен - използва patterns вместо `length`.

---

## Обобщение

| Концепция         | Описание                                      |
| ----------------- | --------------------------------------------- |
| Pattern matching  | Съпоставяне по форма на входа (отгоре надолу) |
| Литерален образец | Точна стойност: `0`, `True`                   |
| Wildcard `_`      | Игнорира стойността                           |
| Guards            | Условия с `\|`, `otherwise`                   |
| Кортежи           | Фиксирани поредици: `(a, b)`, `(a, b, c)`     |
| `fst`, `snd`      | Достъп до елементи на двойка                  |
| Деструктуриране   | Pattern matching на кортежи и списъци         |

# Рекурсия

## Въведение

В Haskell няма `for` или `while` цикли - вместо тях използваме **рекурсия**. Функция извиква сама себе си с „по-малък" аргумент, докато достигне **базов случай**.

---

## 1. Проста рекурсия по числа

### Структура

```haskell
f baseCase = baseResult          -- базов случай
f n        = ... f (n - 1) ...   -- рекурсивна стъпка
```

### Пример: факториел

```haskell
factorial :: Integer -> Integer
factorial 0 = 1
factorial n = n * factorial (n - 1)
```

Как се оценява:

```
factorial 4
= 4 * factorial 3
= 4 * (3 * factorial 2)
= 4 * (3 * (2 * factorial 1))
= 4 * (3 * (2 * (1 * factorial 0)))
= 4 * (3 * (2 * (1 * 1)))
= 24
```

---

## 2. Рекурсия по списъци

Pattern matching на `[]` (празен) и `(x:xs)` (глава + опашка):

```haskell
myLength :: [a] -> Int
myLength []     = 0
myLength (_:xs) = 1 + myLength xs

mySum :: [Int] -> Int
mySum []     = 0
mySum (x:xs) = x + mySum xs
```

---

## 3. Натрупващ параметър (Accumulator)

Вместо да трупаме отложени операции, носим **акумулатор**:

```haskell
-- Без акумулатор (не е tail-recursive)
mySum :: [Int] -> Int
mySum []     = 0
mySum (x:xs) = x + mySum xs

-- С акумулатор (tail-recursive)
mySum' :: [Int] -> Int
mySum' xs = helper 0 xs
  where
    helper acc []     = acc
    helper acc (x:xs) = helper (acc + x) xs
```

Разлика при оценяване:

```
-- Без акумулатор:          -- С акумулатор:
mySum [1,2,3]               mySum' [1,2,3]
= 1 + mySum [2,3]           = helper 0 [1,2,3]
= 1 + (2 + mySum [3])       = helper 1 [2,3]
= 1 + (2 + (3 + mySum []))  = helper 3 [3]
= 1 + (2 + (3 + 0))         = helper 6 []
= 6                          = 6
```

> 💡 Акумулаторната версия не трупа операции в стека - по-ефективна при дълги списъци.

---

## 4. Опашкова рекурсия (Tail Recursion)

Рекурсия е **опашкова**, ако рекурсивното извикване е **последната** операция:

```haskell
-- НЕ е tail-recursive (последната операция е n * ...)
factorial n = n * factorial (n - 1)

-- Tail-recursive (последната операция е самото извикване)
factorial' n = helper 1 n
  where
    helper acc 0 = acc
    helper acc n = helper (acc * n) (n - 1)
```

---

## 5. Рекурсия по числа - числова теория

### Сума на цифрите

```haskell
digitSum :: Int -> Int
digitSum 0 = 0
digitSum n = n `mod` 10 + digitSum (n `div` 10)
```

### Брой цифри

```haskell
countDigits :: Int -> Int
countDigits n
  | n < 10    = 1
  | otherwise = 1 + countDigits (n `div` 10)
```

---

## 6. Помощни функции (Helper pattern)

Когато рекурсията се нуждае от допълнителни параметри, дефинираме помощна функция:

```haskell
-- Обръщане на цифрите на число
reverseNumber :: Int -> Int
reverseNumber n = helper n 0
  where
    helper 0   acc = acc
    helper num acc = helper (num `div` 10) (acc * 10 + num `mod` 10)
```

---

## Обобщение

| Концепция         | Описание                                 |
| ----------------- | ---------------------------------------- |
| Базов случай      | Условие за спиране на рекурсията         |
| Рекурсивна стъпка | Извикване с „по-малък" аргумент          |
| `(x:xs)` pattern  | Деструктуриране на списък                |
| Акумулатор        | Допълнителен параметър за натрупване     |
| Tail recursion    | Рекурсивното извикване е последно        |
| Helper pattern    | Помощна функция с допълнителни параметри |

# Списъци

## Въведение

Списъците са **основната структура от данни** в Haskell. Списъкът е хомогенна, рекурсивно дефинирана, свързана структура - може да съдържа само елементи от **един и същ тип**.

---

## 1. Структура на списък

Списъкът е или:

- `[]` - празен списък
- `x : xs` - елемент `x`, свързан с опашка `xs`

```haskell
-- Тези три записа са еквивалентни:
[1, 2, 3]
1 : [2, 3]
1 : 2 : 3 : []
```

> 💡 `:` (cons) е дясно-асоциативен: `1 : 2 : 3 : [] = 1 : (2 : (3 : []))`.

---

## 2. Основни операции

| Функция   | Тип                        | Описание                 |
| --------- | -------------------------- | ------------------------ |
| `head`    | `[a] -> a`                 | Първи елемент            |
| `tail`    | `[a] -> [a]`               | Всичко без първия        |
| `last`    | `[a] -> a`                 | Последен елемент         |
| `init`    | `[a] -> [a]`               | Всичко без последния     |
| `length`  | `[a] -> Int`               | Дължина                  |
| `null`    | `[a] -> Bool`              | Празен ли е?             |
| `reverse` | `[a] -> [a]`               | Обръщане                 |
| `take`    | `Int -> [a] -> [a]`        | Първите n елемента       |
| `drop`    | `Int -> [a] -> [a]`        | Без първите n            |
| `!!`      | `[a] -> Int -> a`          | Елемент по индекс (от 0) |
| `elem`    | `Eq a => a -> [a] -> Bool` | Съдържа ли се?           |
| `++`      | `[a] -> [a] -> [a]`        | Конкатенация             |

> ⚠️ `head []` и `tail []` хвърлят грешка! Винаги проверявайте за празен списък.

---

## 3. Диапазони (Ranges)

```haskell
ghci> [1..5]          -- [1, 2, 3, 4, 5]
ghci> [1,3..10]       -- [1, 3, 5, 7, 9]
ghci> [10,8..1]       -- [10, 8, 6, 4, 2]
ghci> ['a'..'f']      -- "abcdef"
ghci> [1..]           -- безкраен списък! (Ctrl+C)
```

---

## 4. Pattern matching на списъци

```haskell
-- Празен списък
isEmpty :: [a] -> Bool
isEmpty [] = True
isEmpty _  = False

-- Глава и опашка
myHead :: [a] -> a
myHead (x:_) = x

-- Първи два елемента
firstTwo :: [a] -> (a, a)
firstTwo (x:y:_) = (x, y)
```

---

## 5. Низове като списъци

В Haskell `String = [Char]`, т.е. низовете са списъци от символи:

```haskell
ghci> "hello" !! 0        -- 'h'
ghci> length "hello"      -- 5
ghci> reverse "hello"     -- "olleh"
ghci> 'h' : "ello"        -- "hello"
ghci> words "hello world" -- ["hello", "world"]
ghci> unwords ["a","b"]   -- "a b"
ghci> lines "a\nb"        -- ["a", "b"]
```

---

## 6. Полезни функции

```haskell
ghci> zip [1,2,3] ["a","b","c"]  -- [(1,"a"), (2,"b"), (3,"c")]
ghci> unzip [(1,"a"), (2,"b")]   -- ([1,2], ["a","b"])
ghci> concat [[1,2],[3,4]]       -- [1,2,3,4]
ghci> sum [1,2,3]                -- 6
ghci> product [1,2,3]            -- 6
ghci> maximum [3,1,7]            -- 7
ghci> minimum [3,1,7]            -- 1
ghci> replicate 3 "ha"           -- ["ha","ha","ha"]
```

---

## Обобщение

| Концепция         | Описание                             |
| ----------------- | ------------------------------------ |
| `[]`              | Празен списък                        |
| `x : xs` (cons)   | Елемент + опашка                     |
| `++`              | Конкатенация                         |
| `[a..b]`          | Диапазон                             |
| `head`/`tail`     | Глава/опашка (внимание: не за `[]`!) |
| `take`/`drop`     | Първите n / без първите n            |
| `String = [Char]` | Низовете са списъци                  |
| `zip`/`unzip`     | Обединяване/разделяне на двойки      |

# Функции от по-висок ред

## Въведение

**Функция от по-висок ред** (higher-order function) е функция, която:

- приема **друга функция** като аргумент, и/или
- **връща функция** като резултат.

Това е основен механизъм за абстракция във функционалното програмиране.

---

## 1. Функции като аргументи

```haskell
applyTwice :: (a -> a) -> a -> a
applyTwice f x = f (f x)
```

```haskell
ghci> applyTwice (+3) 10    -- 16
ghci> applyTwice (*2) 5     -- 20
ghci> applyTwice reverse [1,2,3]  -- [1,2,3]
```

> 💡 `(a -> a)` е типът на функция от `a` в `a`. Скобите са задължителни в сигнатурата.

---

## 2. `map`

`map` прилага функция върху **всеки елемент** от списък:

```haskell
map :: (a -> b) -> [a] -> [b]
map _ []     = []
map f (x:xs) = f x : map f xs
```

```haskell
ghci> map (*2) [1, 2, 3]        -- [2, 4, 6]
ghci> map even [1, 2, 3, 4]     -- [False, True, False, True]
ghci> map show [1, 2, 3]        -- ["1", "2", "3"]
ghci> map length ["hi", "world"] -- [2, 5]
```

---

## 3. `filter`

`filter` оставя само елементите, за които предикатът е `True`:

```haskell
filter :: (a -> Bool) -> [a] -> [a]
filter _ []     = []
filter p (x:xs)
  | p x       = x : filter p xs
  | otherwise = filter p xs
```

```haskell
ghci> filter even [1..10]        -- [2, 4, 6, 8, 10]
ghci> filter (>3) [1, 5, 2, 8]  -- [5, 8]
ghci> filter (/=' ') "hello world"  -- "helloworld"
```

---

## 4. `foldr` (дясно свиване)

`foldr` **свива** списък отдясно наляво, заместващ всеки `:` с операция и `[]` с начална стойност:

```haskell
foldr :: (a -> b -> b) -> b -> [a] -> b
foldr _ acc []     = acc
foldr f acc (x:xs) = f x (foldr f acc xs)
```

Визуализация за `foldr (+) 0 [1, 2, 3]`:

```
  :           +
 / \         / \
1   :   →   1   +
   / \         / \
  2   :       2   +
     / \         / \
    3  []       3   0
```

```haskell
ghci> foldr (+) 0 [1, 2, 3]      -- 6
ghci> foldr (:) [] [1, 2, 3]     -- [1, 2, 3]  (идентичност!)
ghci> foldr (++) "" ["a","b","c"] -- "abc"
```

---

## 5. `foldl` (ляво свиване)

`foldl` свива списък **отляво надясно**, натрупвайки резултат в акумулатор:

```haskell
foldl :: (b -> a -> b) -> b -> [a] -> b
foldl _ acc []     = acc
foldl f acc (x:xs) = foldl f (f acc x) xs
```

Визуализация за `foldl (+) 0 [1, 2, 3]`:

```
foldl (+) 0 [1,2,3]
= foldl (+) (0+1) [2,3]
= foldl (+) ((0+1)+2) [3]
= foldl (+) (((0+1)+2)+3) []
= ((0+1)+2)+3
= 6
```

```haskell
ghci> foldl (+) 0 [1, 2, 3]    -- 6  (същото като foldr за (+))
ghci> foldl (-) 10 [1, 2, 3]   -- 4  (((10-1)-2)-3)
ghci> foldr (-) 0 [1, 2, 3]    -- 2  (1-(2-(3-0)))  ← РАЗЛИЧНО!
```

> ⚠️ За **некомутативни** операции (като `-`), `foldl` и `foldr` дават **различни** резултати!

---

## 6. `foldr` vs `foldl`

| Свойство            | `foldr`                         | `foldl`                 |
| ------------------- | ------------------------------- | ----------------------- |
| Посока              | отдясно наляво                  | отляво надясно          |
| Рекурсия            | обикновена                      | опашкова (с акумулатор) |
| Безкрайни списъци   | ✅ работи (ако `f` е мързелива) | ❌ не работи            |
| Пренаписване на `:` | `foldr (:) [] xs == xs`         | -                       |
| Предпочитание       | за конструиране на списъци      | за числови агрегати     |

> 💡 Правило: ако резултатът е **списък** → `foldr`. Ако е **число/акумулатор** → `foldl'` (стриктна версия).

## 7. Currying и частично прилагане

Всяка функция в Haskell приема **точно един** аргумент. Функция с повече аргументи е верига от функции:

```haskell
add :: Int -> Int -> Int
add x y = x + y
-- Еквивалентно на: add :: Int -> (Int -> Int)
-- add 3 :: Int -> Int  (частично приложена)
```

```haskell
ghci> map (+ 3) [1, 2, 3]       -- [4, 5, 6]
ghci> map (* 2) [1, 2, 3]       -- [2, 4, 6]
ghci> filter (> 5) [3, 7, 1, 9] -- [7, 9]
ghci> map (take 2) ["hello", "world"]  -- ["he", "wo"]
```

---

## Обобщение

| Функция            | Какво прави                    | Типична употреба      |
| ------------------ | ------------------------------ | --------------------- |
| `map f xs`         | Трансформира всеки елемент     | `map (*2) [1,2,3]`    |
| `filter p xs`      | Филтрира по предикат           | `filter even [1..10]` |
| `foldr f z xs`     | Свива отдясно                  | `foldr (+) 0 [1,2,3]` |
| `foldl f z xs`     | Свива отляво                   | `foldl (+) 0 [1,2,3]` |
| Частично прилагане | `f x` когато `f` очаква 2 арг. | `map (+3) xs`         |

# Ламбди, композиция и list comprehensions

## 1. Ламбда функции

Анонимна (безименна) функция, дефинирана на място:

```haskell
\x -> x + 1          -- функция, която прибавя 1
\x y -> x + y        -- функция на два аргумента
\(x, y) -> x + y     -- деструктуриране на кортеж
```

```haskell
ghci> (\x -> x * 2) 5           -- 10
ghci> map (\x -> x ^ 2) [1..5]  -- [1, 4, 9, 16, 25]
ghci> filter (\x -> x `mod` 3 == 0) [1..20]  -- [3, 6, 9, 12, 15, 18]
```

> 💡 Ламбда функциите са удобни когато ни трябва кратка, еднократна функция, без да й даваме име.

---

## 2. Секции на оператори

Всеки инфиксен оператор може да се превърне в частично приложена функция:

```haskell
(+ 3)    -- \x -> x + 3
(* 2)    -- \x -> x * 2
(> 5)    -- \x -> x > 5
(3 :)    -- \xs -> 3 : xs
(++ "!")  -- \s -> s ++ "!"
```

```haskell
ghci> map (+ 10) [1, 2, 3]       -- [11, 12, 13]
ghci> filter (< 5) [3, 7, 1, 9]  -- [3, 1]
ghci> map (++ "!") ["hi", "yo"]  -- ["hi!", "yo!"]
```

> ⚠️ За изваждане внимавайте: `(- 3)` е **числото** -3, не секция! Използвайте `subtract 3` вместо.

---

## 3. Композиция на функции

Операторът `(.)` комбинира две функции: първо прилага дясната, после лявата.

```haskell
(.) :: (b -> c) -> (a -> b) -> a -> c
(f . g) x = f (g x)
```

```haskell
ghci> ((*2) . (+3)) 5    -- (*2) (5+3) = 16
ghci> (head . reverse) [1,2,3]  -- last [1,2,3] = 3
ghci> map (negate . abs) [-3, 2, -5]  -- [-3, -2, -5]
```

Композицията е полезна за **безточков стил** (point-free style):

```haskell
-- С аргумент:
doubleAll xs = map (*2) xs

-- Безточков:
doubleAll = map (*2)

-- С аргумент:
sumOfEvens xs = sum (filter even xs)

-- Безточков:
sumOfEvens = sum . filter even
```

## 4. `curry`, `uncurry` и `flip`

```haskell
curry   :: ((a, b) -> c) -> a -> b -> c    -- кортежна → curry
uncurry :: (a -> b -> c) -> (a, b) -> c    -- curry → кортежна
flip    :: (a -> b -> c) -> b -> a -> c    -- разменя аргументите
```

```haskell
ghci> uncurry (+) (3, 4)           -- 7
ghci> map (uncurry (+)) [(1,2), (3,4)]  -- [3, 7]
ghci> flip (:) [1,2] 3            -- [3,1,2]
ghci> flip div 10 100             -- 10  (100 `div` 10)
```

---

## 5. Оператор `$`

`$` е оператор за прилагане на функция с **най-нисък приоритет** - позволява избягване на скоби:

```haskell
($) :: (a -> b) -> a -> b
f $ x = f x
```

```haskell
-- Без $:
sum (filter even (map (*2) [1..10]))

-- С $:
sum $ filter even $ map (*2) [1..10]
```

---

## 6. List Comprehensions

**List comprehensions** са синтактична захар за комбинация от `map` и `filter`:

```haskell
[<израз> | <генератор>, <условие>, ...]
```

```haskell
ghci> [x * 2 | x <- [1..5]]            -- [2, 4, 6, 8, 10]
ghci> [x | x <- [1..20], even x]       -- [2, 4, 6, ..., 20]
ghci> [x ^ 2 | x <- [1..10], odd x]    -- [1, 9, 25, 49, 81]
```

### Множество генератори

```haskell
ghci> [(x, y) | x <- [1..3], y <- ['a'..'c']]
-- [(1,'a'),(1,'b'),(1,'c'),(2,'a'),(2,'b'),(2,'c'),(3,'a'),(3,'b'),(3,'c')]

ghci> [x * y | x <- [1..3], y <- [10,20]]
-- [10, 20, 20, 40, 30, 60]
```

### Зависими генератори

```haskell
ghci> [(x, y) | x <- [1..5], y <- [x..5]]
-- [(1,1),(1,2),...,(1,5),(2,2),(2,3),...,(5,5)]
```

### Let в comprehension

```haskell
ghci> [(name, avg) | (name, grades) <- students
                   , let avg = sum grades `div` length grades
                   , avg >= 5]
```

---

## 7. Еквивалентност: comprehension ↔ map/filter

```haskell
-- Comprehension:
[x * 2 | x <- xs, even x]

-- Еквивалентно на:
map (* 2) (filter even xs)
```

```haskell
-- Comprehension:
[(x, y) | x <- xs, y <- ys]

-- Еквивалентно на:
concatMap (\x -> map (\y -> (x, y)) ys) xs
```

## Обобщение

| Концепция         | Синтаксис             | Пример                         |
| ----------------- | --------------------- | ------------------------------ |
| Ламбда            | `\x -> ...`           | `\x -> x + 1`                  |
| Секция            | `(op val)`            | `(+3)`, `(>0)`                 |
| Композиция        | `f . g`               | `sum . filter even`            |
| `$`               | `f $ x`               | `sum $ map (*2) xs`            |
| `curry`/`uncurry` | -                     | `uncurry (+) (3,4)`            |
| `flip`            | -                     | `flip div 10 100`              |
| Comprehension     | `[e \| x <- xs, p x]` | `[x^2 \| x <- [1..10], odd x]` |
